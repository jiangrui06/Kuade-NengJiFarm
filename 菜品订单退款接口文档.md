# 菜品订单退款接口文档

## 接口信息

| 项目 | 内容 |
|---|---|
| **接口名称** | 菜品订单退款 |
| **接口地址** | `POST /api/dish/order/refund` |
| **接口说明** | 后台管理员对菜品/点餐订单执行退款操作（含微信支付退款） |
| **认证方式** | Bearer Token（管理员登录 token） |

---

## 请求参数

### 请求头

| 参数 | 值 | 必填 |
|---|---|---|
| `Authorization` | `Bearer {token}` | 是 |
| `Content-Type` | `application/json` | 是 |

### 请求体（JSON）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `orderId` | long | 是 | 菜品订单ID（主键） |
| `refundReason` | string | 否 | 退款原因说明 |

### 请求示例

```json
{
    "orderId": 262,
    "refundReason": "用户要求退款"
}
```

---

## 响应参数

### 成功响应

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | int | 200 |
| `message` | string | "退款成功" |
| `data.refundId` | string | 退款编号（格式：`RF{时间戳}{随机3位}`） |
| `data.orderId` | string | 订单号（平台订单号） |
| `data.refundAmount` | string | 退款金额（两位小数） |
| `data.refundTime` | string | 退款时间（`yyyy-MM-dd HH:mm`） |
| `data.operator` | string | 操作人账号 |

### 成功示例

```json
{
    "code": 200,
    "message": "退款成功",
    "data": {
        "refundId": "RF20260522153000123456",
        "orderId": "DISH20260522115821632680",
        "refundAmount": "0.01",
        "refundTime": "2026-05-22 15:30",
        "operator": "admin"
    }
}
```

## 错误码

| HTTP 状态码 | code | message | 说明 |
|---|---|---|---|
| 401 | - | 未返回 | 登录已过期，请重新登录 |
| 200 | 400 | 请求参数不完整：orderId 不能为空 | 未传 orderId |
| 200 | 400 | 订单不存在或已被删除 | orderId 无效 |
| 200 | 400 | 该订单已完成退款，请勿重复操作 | 已退款订单再次操作 |
| 200 | 400 | 该订单已完成或已取消，无法退款 | 订单状态为已完成(3)或已取消(4) |
| 200 | 400 | 当前订单状态不允许退款 | 订单状态不为待支付(1)或备餐中(2) |
| 200 | 400 | 微信退款失败：{具体原因} | 微信退款接口异常 |

### 错误示例

```json
{
    "code": 400,
    "message": "该订单已完成退款，请勿重复操作"
}
```

## 业务逻辑说明

### 流程图

```
请求 → 验证Token → 校验参数
         ↓
    查找订单（orderId）
         ↓
    幂等性检查（是否已有退款记录，order_type = "food"）
         ↓
    订单状态校验（仅 status=1 或 2 可退）
         ↓
    微信退款调用（非MOCK订单）
         ↓
    写入退款记录（refund_record，order_type = "food"）
         ↓
    更新订单状态为已退款（status = 4）
         ↓
    返回退款结果
```

### 可退款的订单状态

| 状态ID | 状态名称 | 可退款 |
|---|---|---|
| 1 | 待付款 / 备餐中 | ✅ |
| 2 | 备餐中（已支付） | ✅ |
| 3 | 已完成 | ❌ |
| 4 | 已取消 / 已退款 | ❌ |

### 退款记录说明

- 写入 `refund_record` 表
- `order_type = "food"`
- `status = "completed"`（后台直接退款完成）
- `reason = "admin_refund"`（标识为管理员操作）

## 与卷类/产品退款接口的差异

| 对比项 | 菜品退款 | 产品退款 | 卷类退款 |
|---|---|---|---|
| 路由 | `api/dish/order/refund` | `api/product/order/refund` | `api/activity-order/refund` |
| order_type | `"food"` | `"goods"` | `"activity"` |
| 退款后状态 | 4（已退款） | 7（已退款） | 4（已退款） |
| 恢复库存 | 不需要 | ✅ 恢复 `in_stock` | 不需要 |
| 可退状态 | 1, 2 | 2, 3 | 2, 3 |
| 驳回退款 | ❌ | ❌ | ✅ |
