# 用户禁用状态 - 接口文档

## 概述

role 和 status 分離：

| 字段 | 含义 | 说明 |
|------|------|------|
| `role` | 用户身份 | 普通用户/员工/管理员，禁用后不变 |
| `status` | 账号状态 | `active` = 正常，`disabled` = 已禁用 |

禁用状态存 `sys_config` 表（key = `disabled_user_{userId}`），不再改 `role_id`。

---

## 1. 禁用用户

`PUT /api/back-user/{userId}/disable`

**Request:**
```
PUT /api/back-user/123/disable
Authorization: Bearer <admin-token>
```

**Response (200):**
```json
{
  "code": 200,
  "message": "用户已禁用"
}
```

**行为变化的字段：** `role_id` 不変，status 写 sys_config。

---

## 2. 启用用户

`PUT /api/back-user/{userId}/enable`

**Request:**
```
PUT /api/back-user/123/enable
Authorization: Bearer <admin-token>
```

**Response (200):**
```json
{
  "code": 200,
  "message": "用户已启用"
}
```

**行为变化的字段：** 删 sys_config 记录，`role_id` 恢复为原始值。

## 3. 用户列表

`GET /api/back-user/list`

**Request:**
```
GET /api/back-user/list?keyword=&pageNum=1&pageSize=20
Authorization: Bearer <admin-token>
```

**Response (200):**
```json
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": "1",
```

| `status` | string | `active` = 正常，`disabled` = 已禁用 |

## 4. 微信登录

`POST /api/auth/wxlogin`

**Request:**
```json
{
  "code": "wx-login-code",
  "avatar": "https://..."
}
```

**Response (403 - 已禁用):**
```json
{
  "code": 403,
  "message": "账号已禁用，请联系管理员"
}
```

**行为变化的字段：** 禁用校验从 `user.RoleId == disabledRoleId` 改为查 `sys_config`。
