# 农场优选分页问题 — 前端修改方案

## 问题现象

农场优选页面加载后只显示 10 条商品，往下滑动到底部显示"已加载全部商品"，但数据库中有更多已上架商品。

## 根因

前端 `farm-goods.js` 采用"一次拉取 + 本地切片"模式：

1. `onLoad()` → `getGoodsList()` 调用 `api.farmGoods.getList({ type: 'all' })`，**不带 page/pageSize 参数**
2. 后端 `pageSize` 默认值为 10 → 只返回第 1 页的 10 条
3. `filterGoods()` 只在这 10 条里做本地过滤
4. `onReachBottom()` **是空函数**（L323），滑到底部不触发任何加载

## 需要修改的文件

| 文件 | 改动量 |
|---|---|
| `user-pages/farm-goods/farm-goods.js` | ~50 行 |
| `user-pages/farm-goods/farm-goods.wxml` | ~5 行 |
| **后端无需修改** | 0 行 |

## farm-goods.js 修改

### 1. data 中新增字段

在 `data: { ... }` 中添加以下字段：

```js
page: 1,
pageSize: 20,
hasMore: true,
allLoaded: false,
allGoodsList: [],  // 存放所有已加载商品（完整数据池）
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `page` | number | 当前已加载的页码 |
| `pageSize` | number | 每页请求数量 |
| `hasMore` | boolean | 是否还有下一页 |
| `allLoaded` | boolean | 全部加载完成（控制底部提示文字） |
| `allGoodsList` | array | 累积所有已加载的商品原始数据 |

### 2. 拆分加载逻辑：新增 loadGoodsPage() 方法

将 `getGoodsList()` 中调用 API 并处理数据的逻辑抽取为 `loadGoodsPage(page, reset)`：

```
getGoodsList()
  └── 重置 page=1, hasMore=true, allLoaded=false
      └── 调用 loadGoodsPage(1, true)

loadGoodsPage(page, reset)
  ├── 显示加载中提示
  ├── 调用 api.farmGoods.getList({ type: 'all', page, pageSize })
  ├── 计算 hasMore = page * pageSize < total
  ├── if (reset) → 替换 allGoodsList
  ├── else       → 追加到 allGoodsList
  ├── 更新 page, hasMore, allLoaded
  └── 调用 filterGoods() 重新过滤显示
```

**关键逻辑**：

```js
loadGoodsPage: function (page, reset = false) {
  const that = this;
  wx.showLoading({ title: '加载中...' });

  api.farmGoods.getList({ type: 'all', page, pageSize: this.data.pageSize })
    .then(data => {
      const list = data.list || [];
      const total = data.total || 0;
      const pageSize = data.pageSize || this.data.pageSize;
      const hasMore = page * pageSize < total;

      // 格式化商品数据（处理 image、price、tags 等）
      const processedGoods = list.filter(item => item.type !== 'acre').map(item => ({
        ...item,
        id: String(item.id),
        image: this.processImageUrl(item.image),
        price: typeof item.price === 'string' ? item.price.replace(/[¥￥]/g, '') : item.price,
        originalPrice: typeof item.originalPrice === 'string' ? item.originalPrice.replace(/[¥￥]/g, '') : item.originalPrice,
        tags: item.tags || [],
        stock: item.stock || 0
      }));

      const processedAcre = list.filter(item => item.type === 'acre').map(item => ({
        ...item,
        id: String(item.id),
        image: this.processImageUrl(item.image || item.cover),
        price: typeof item.price === 'string' ? item.price.replace(/[¥￥]/g, '') : item.price,
        tags: item.tags || [],
        stock: item.stock || 0
      }));

      if (reset) {
        that.setData({
          goodsList: processedGoods,
          acreList: processedAcre,
          page,
          hasMore,
          allLoaded: !hasMore,
          loading: false
        }, () => that.filterGoods());
      } else {
        that.setData({
          goodsList: [...that.data.goodsList, ...processedGoods],
          acreList: [...that.data.acreList, ...processedAcre],
          page,
          hasMore,
          allLoaded: !hasMore,
          loadingMore: false
        }, () => that.filterGoods());
      }
    })
    .catch(err => {
      that.setData({ loading: false, loadingMore: false });
      wx.showToast({ title: '加载失败', icon: 'none' });
    })
    .finally(() => wx.hideLoading());
},
```

### 3. 修改 getGoodsList()

改为重置状态后调用 `loadGoodsPage`：

```js
getGoodsList: function () {
  this.setData({ loading: true, page: 1, hasMore: true, allLoaded: false });
  this.loadGoodsPage(1, true);
},
```

### 4. 修改 onReachBottom()

从空函数改为加载下一页：

```js
onReachBottom: function () {
  if (!this.data.hasMore || this.data.loadingMore) return;

  this.setData({ loadingMore: true });
  this.loadGoodsPage(this.data.page + 1, false);
},
```

### 5. filterGoods() 无需修改

`filterGoods()` 已经基于 `this.data.goodsList` 和 `this.data.acreList` 做本地过滤，追加数据后仍然兼容，无需改动。

## farm-goods.wxml 修改

### loading-more 区域

当前代码（L67-70）：

```html
<!-- 加载更多 -->
<view class="loading-more" wx:if="{{loadingMore}}">
  <text>加载中...</text>
</view>
```

改为显示两种状态：

```html
<!-- 加载更多 -->
<view class="loading-more" wx:if="{{loadingMore}}">
  <text>加载中...</text>
</view>
<view class="loading-more" wx:elif="{{allLoaded && (goodsList.length > 0 || acreList.length > 0)}}">
  <text>已加载全部商品</text>
</view>
```

| 状态 | 显示文字 |
|---|---|
| `loadingMore = true` | "加载中..." |
| `allLoaded = true` | "已加载全部商品" |
| 首次加载（loading=true） | 不显示底部提示 |
| 正常有更多数据 | 不显示底部提示 |

## 数据流总结

```
onLoad()
  └── getGoodsList()
        └── loadGoodsPage(1, true)          ← 拉取第 1 页，替换数据
              ├── api.farmGoods.getList()
              ├── 处理商品数据
              ├── goodsList / acreList = 第 1 页数据
              └── filterGoods()              ← 本地过滤展示

用户下滑到底
  └── onReachBottom()
        └── if hasMore → loadGoodsPage(2, false)  ← 拉取第 2 页，追加数据
              ├── api.farmGoods.getList()
              ├── 处理商品数据
              ├── goodsList / acreList += 第 2 页数据
              └── filterGoods()              ← 重新过滤展示

用户再次下滑到底
  └── onReachBottom()
        └── ...循环，直到 hasMore = false
              └── 底部显示"已加载全部商品"
```

## 验证标准

1. 农场优选页面加载后显示超过 10 条商品（pageSize=20 条）
2. 下滑到触发 `onReachBottom`，加载下一页并追加展示
3. 所有数据加载完后底部显示"已加载全部商品"
4. 分类切换、关键词搜索、价格筛选在已加载数据范围内正常工作
5. 首次加载显示 loading，后续分页显示 "加载中..."
