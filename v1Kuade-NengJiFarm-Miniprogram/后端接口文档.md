## 1. 通用约定

### 1.1 认证方式

```
Authorization: Bearer <token>
```

Token 通过微信登录获取，存储在 `wx.getStorageSync('token')`。

**需要登录的接口前缀**（无 token 时自动跳转登录页）：

```
/api/user  /api/orders  /api/cart  /api/OrderDetails  /api/order/create
/api/commodity-order  /api/pay  /api/acres  /api/address  /api/logistics
/api/staff  /api/activity  /api/points
```

### 1.2 通用响应格式

**成功：**
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**失败（部分接口返回 HTTP 非 200，部分返回 HTTP 200 + 业务 code）：**
```json
{
  "code": 400,
  "message": "参数错误"
}
```

### 1.3 通用错误码

| code | HTTP 状态码 | 说明 |
|------|------------|------|
| 0 / 200 | 200 | 成功 |
| 400 | 200 | 参数错误/业务错误（如核销失败） |
| 401 | 401 | 未登录 |
| 403 | 403 | 权限不足/已过期 |
| 404 | 404 | 资源不存在 |
| 409 | 200 | 冲突（如重复核销） |
| 500 | 500 | 服务器错误 |

> **注意**：核销类接口所有错误以 HTTP 200 + 业务 code 形式返回，前端根据 `res.data.message` 显示提示

### 1.4 分页参数

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| page | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页条数 |

**分页响应：**
```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 1.5 前端调用方式

前端统一在 `utils/api.js` 中封装，按模块导出：

```js
const api = require('../../utils/api');

// GET 请求
api.home.getData({ page: 1 })

// POST 请求
api.order.createCommodityV2({ items: [...], deliveryMethod: 'express' })

// 上传文件
api.file.upload(filePath, { path: 'avatar' })
```

## 2. 认证模块 (`api/auth`)

### 2.1 微信登录

```
POST /api/Auth/wxlogin
```

**请求体：**
```json
{
  "code": "wx_login_code",
  "nickName": "用户昵称",
  "avatarUrl": "https://..."
}
```

**响应：**
```json
{
  "code": 0,
  "data": {
    "token": "jwt_token_string",
    "userId": 1,
    "role": "user"
  }
}
```

**前端调用：** `api.auth.wxLogin(code, userInfo)`

### 2.2 手机号登录

```
POST /api/Auth/wx-phone-login
```

**请求体：**
```json
{
  "code": "wx_login_code",
  "phoneCode": "wx_phone_code",
  "avatar": "https://...",
  "nickname": "用户昵称"
}
```

**响应：**
```json
{
  "code": 0,
  "data": {
    "token": "jwt_token_string",
    "userId": 1,
    "role": "user",
    "phone": "138****8000"
  }
}
```

**前端调用：** `api.auth.phoneLogin(data)`

### 2.3 检查登录态

```
GET /api/auth/check
```

**Headers：** `Authorization: Bearer <token>`

**响应：**
```json
{
  "code": 0,
  "data": {
    "userId": 1,
    "role": "user",
    "nickname": "张三"
  }
}
```

**前端调用：** 登录后自动携带 token

### 2.4 获取手机号（已登录补充）

```
POST /api/auth/phone
```

**Headers：** `Authorization: Bearer <token>`

**请求体：**
```json
{
  "code": "wx_phone_code"
}
```

**前端调用：** 登录后用户授权手机号时调用

### 2.5 健康检查

```
GET /api/auth/diagnostics
```

无需登录，返回后端服务状态。**前端通常不调用**，仅调试用。

## 6. 订单 (`api/orders` / `api/order` / `api/OrderDetails` / `api/commodity-order`)

订单模块有多个控制器，分工如下：

| 控制器 | 路由前缀 | 说明 |
|--------|----------|------|
| OrdersController | `/api/orders` | **统一订单管理**（聚合查询所有类型订单，主接口） |
| OrderController | `/api/order` | 旧版订单 CRUD，含点餐菜单 |
| OrderDetailsController | `/api/OrderDetails` | 聚合订单创建 |
| CommodityOrderController | `/api/commodity-order` | 商品订单创建（支持配送方式） |

### 6.1 创建商品订单（推荐）

```
POST /api/commodity-order/create
```

**请求体：**
```json
{
  "items": [
    { "id": 1, "price": 29.9, "quantity": 2 }
  ],
  "deliveryMethod": "express",
  "addressId": 1
}
```

**配送方式：**

| deliveryMethod | 说明 | addressId |
|----------------|------|-----------|
| `express` | 快递配送 | 必填 |
| `pickup` | 到店自取 | 不传 |

**响应：**
```json
{
  "code": 0,
  "data": {
    "orderNo": "ORDER202605170001",
    "orderId": "order_001",
    "orderNumber": "ORDER202605170001",
    "id": "order_001"
  }
}
```

> 四个字段指代同一值，前端按优先级取第一个非空

**前端调用：** `api.order.createCommodityV2(data)`

### 6.2 统一订单列表（主接口）

### 7.2 创建微信支付（JSAPI）

```
POST /api/pay/jsapi
```

**请求体：**
```json
{
  "orderNo": "ORDER202605170001",
  "orderId": "order_001",
  "id": "order_001",
```

**响应（微信小程序调起支付所需参数）：**
```json
{

    "paySign": "signature_string",

```

**前端调用：**
```js
api.pay.createJsapi(data).then(res => {
  wx.requestPayment({
    timeStamp: res.timeStamp,
    nonceStr: res.nonceStr,
    package: res.package,
    signType: res.signType,
    paySign: res.paySign,
    success: () => { /* 支付成功 */ }
  });
});
```

## 17. 状态流转图

### 17.1 商品订单状态

```
快递配送：
  pending（待付款） → paid（已付款） → shipping（配送中） → completed（已完成）
       │                   │
       └── cancelled ──────┘

到店自取：
  pending（待付款） → paid（已付款） → verify_pending（待核销） → verified（已核销）
       │                   │
       └── cancelled ──────┘
```

### 17.2 活动订单状态

```
pending（待付款） → paid（已报名） → verified（已核销）
       │
       └── cancelled
```

### 17.3 积分兑换订单状态

```
pending（待核销） → verified（已核销）
       │
       └── cancelled（已取消）
```

### 17.4 退款状态

```
paid / shipping / completed → refunding（退款中） → refunded（已退款）
                                                └── 取消退款 → 恢复原状态
```

### 17.5 订单状态对应表

| status | statusText | 说明 | 适用订单类型 |
|--------|-----------|------|-------------|
| `pending` | 待付款 | 已创建未支付 | goods / food / activity |
| `paid` | 已付款 | 已支付待处理 | goods（快递待发货）/ food（厨房制作中） |
| `shipping` | 配送中 | 已发货 | goods（快递） |
| `completed` | 已完成 | 已签收/已确认 | goods / food |
| `cancelled` | 已取消 | 已取消 | goods / food / activity |
| `verify_pending` | 待核销 | 待核销 | goods（自取）/ activity |
| `verified` | 已核销 | 已核销 | goods（自取）/ activity |
| `refunding` | 退款中 | 退款处理中 | goods |
| `refunded` | 已退款 | 已退款 | goods |
