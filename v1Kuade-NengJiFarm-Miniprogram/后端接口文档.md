## 1. 通用约定

### 1.1 认证方式

`Authorization: Bearer <token>`

Token 通过微信登录获取，存储在 `wx.getStorageSync('token')`。

以下新增接口需要登录态：

```
/api/points  /api/exchange  /api/user/address  /api/commodity-order
/api/orders  /api/staff-verify
```

### 1.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0 成功，非 0 失败 |
| message | string | 提示信息 |
| data | object/array | 响应数据 |

### 1.3 通用错误码

| code | 说明 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录/无权限 |
| 403 | 权限不足/已过期 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 1.4 分页参数

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| page | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页条数 |

**分页响应：**

```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 2.2 创建商品订单（支持配送方式）

```
POST /api/commodity-order/create
```

**请求体：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| items | array | 是 | 商品项列表 |
| deliveryMethod | string | 是 | `express` 快递 / `pickup` 到店自取 |
| addressId | int | 快递必填 | 收货地址 ID |

**items 项：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 是 | 商品 ID |
| price | number | 是 | 单价 |
| quantity | int | 是 | 数量 |

**请求示例（快递）：**

```json
{
  "items": [
    { "id": 1, "price": 29.9, "quantity": 2 }
  ],
  "deliveryMethod": "express",
  "addressId": 1
}
```

**请求示例（自取）：**

```json
{
  "items": [
    { "id": 1, "price": 29.9, "quantity": 2 }
  ],
  "deliveryMethod": "pickup"
}
```

**响应 data：**

```json
{
  "orderNo": "ORDER202605170001",
  "orderId": "order_001",
  "orderNumber": "ORDER202605170001",
  "id": "order_001"
}
```

> orderNo / orderId / orderNumber / id 四字段兼容，前端按优先级取第一个非空值

**前端调用：** confirm-order.js `createOrder()` → `api.order.createCommodityV2(payload)`

|------|------|------|------|
| code | string | 是 | 核销码（二维码内容） |

**响应 data：**

{
  "verified": true,
  "alreadyVerified": false,
  "voucherType": "goods_pickup",
  "type": "goods_pickup",

## 5. 数据字典

### 5.1 配送方式

| deliveryMethod | 说明 | 适用场景 |
|----------------|------|----------|
| express | 快递配送 | 需填写收货地址 |
| pickup | 到店自取 | 出示核销码 |

### 5.2 订单状态（新增/变更）

| status | 说明 | 适用 |
|--------|------|------|
| verify_pending | 待核销 | 活动订单、自取商品 |
| verified | 已核销 | 活动订单、自取商品 |

**商品订单状态流转：**

```
快递配送：pending → paid → shipping → completed
到店自取：pending → paid → verify_pending → verified
```

### 5.3 核销类型

| voucherType / type | 说明 | 显示名 |
|--------------------|------|--------|
| goods_pickup | 商品自取核销 | 商品自取 |
| pick | 采摘券核销 | 采摘券 |
| 其他 | 活动券核销 | 活动券 |

### 5.4 积分类型

| type | 说明 | 前端显示 |
|------|------|----------|
| earn | 获得积分 | + |
| spend | 消费积分 | - |

### 5.5 积分换算规则

- 消费 **10 元** = **1 积分**，向下取整
- 获取条件：在场消费过的用户
- 后续扩展：可配置、会员等级差异化
