## 一、项目概述

「酷德能量农场」是一个**农业电商 + 体验式认购**微信小程序，面向终端用户和农场员工两类角色：

- **普通用户**：浏览农场商品、参与活动、认购一亩田、下单购物、查看物流
- **员工用户**：扫码核销采摘券/活动券，管理核销记录

小程序采用 **自定义 TabBar（custom-tab-bar）** 实现角色差异化导航——用户看到 4 个 Tab（首页 / 活动 / 购物车 / 我的），员工只看到 2 个 Tab（工作台 / 我的），登录时根据 `user_role` 字段自动切换。

## 三、核心模块说明

### 3.1 用户体系 & 登录

| 项目 | 说明 |
|------|------|
| **登录方式** | 微信 `wx.login()` 获取 code → 后端换 `session_key` → `getPhoneNumber` 获取手机号 |
| **接口** | `POST /api/Auth/wx-phone-login` |
| **角色标识** | `user_role` 存入 `Storage`，值：`user`（普通用户）/ `staff`（员工） |
| **Token** | 登录成功后返回 `token`，存入 `Storage`，后续请求通过 `Authorization: Bearer <token>` 携带 |
| **登录拦截** | `api.js` 中对需认证接口做统一校验，无 token 时自动跳转登录页 |

### 3.2 自定义 TabBar（角色切换）

```
普通用户 TabBar：     首页 │ 活动 │ 购物车 │ 我的
员工用户 TabBar：     工作台 │ 我的
```

- `custom-tab-bar/index.js` 的 `init()` 方法读取 `user_role` 决定渲染哪套 tab 配置
- 员工工作台位于分包内（非 TabBar 页面），使用 `redirectTo` 切换而非 `switchTab`

### 3.3 商城模块

| 功能 | 页面 | 关键接口 |
|------|------|---------|
| 商品列表/搜索 | farm-goods, search | `GET /api/goods`, `/api/goods/search` |
| 商品详情 | goods-detail | `GET /api/goods/{id}` |
| 加入购物车 | goods-detail, buy | `POST /api/cart/add` |
| 购物车管理 | cart | `GET/PUT/DEL /api/cart` |
| 确认下单 | confirm-order | `POST /api/OrderDetails/create` |
| 支付 | pay | `POST /api/pay/jsapi`, `/api/pay/initiate-payment` |

### 3.4 订单模块

| 状态 | 说明 |
|------|------|
| `pending` | 待付款（含 15 分钟倒计时，超时自动取消） |
| `paid` | 已付款 |
| `shipping` | 配送中 |
| `completed` | 已完成 |
| `cancelled` | 已取消（仅显示商品信息 + 价格 + 编号 + 取消时间） |

**特殊能力：**
- **订单计时器**（`order-timer.js`）：待付款订单倒计时，超时自动调用取消接口并本地标记
- **已取消订单差异化展示**：隐藏收货地址和物流信息，订单信息区只保留编号和取消时间
- **三种订单类型**：普通商城订单（`order`）、活动订单（`activity`）、认购订单（`acre`）

### 3.5 活动模块

- 活动列表 → 活动详情 → 报名注册
- 支持**核销二维码**生成，用于现场签到/验券

### 3.6 认购一亩田

- 用户可认购农田地块，查看认购详情
- 属于独立订单类型，走专门的 `/api/acres` 接口系列

### 3.7 员工核销模块

| 页面 | 功能 |
|------|------|
| staff-home | 工作台首页，扫码入口（调用 `wx.scanCode`） |
| staff-verify | 展示券码详情，执行核销操作（`POST /api/staff/verify`） |

## 四、API 接口总览

所有接口封装在 `utils/api.js`，共 **8 大模块、40+ 接口**：

| 模块 | 前缀 | 主要功能 |
|------|------|---------|
| **home** | `/api/home` | 首页数据聚合 |
| **goods** | `/api/goods` | 商品 CRUD + 搜索 |
| **activity** | `/api/activity` | 活动列表 / 详情 / 报名 |
| **acre** | `/api/acres` | 一亩田认购 |
| **order** | `/api/orders` | 订单全生命周期管理（创建/查询/取消/删除/支付/二维码） |
| **cart** | `/api/cart` | 购物车增删改查 |
| **pay** | `/api/pay` | 微信 JSAPI 支付 / 支付状态查询 |
| **logistics** | `/api/logistics` | 物流轨迹查询 |
| **staff** | `/api/staff` | 员工扫码核销 |
| **user** | `/api/user` | 用户信息 / 地址管理 |
| **file** | `/api/file` | 图片上传下载 |

**请求规范：**
- 认证方式：`Authorization: Bearer <token>`
- 响应格式：`{ code: 0, data: {...} }`（code=0 为成功）
- 无 token 访问受保护接口时自动跳转登录页

## 五、关键技术实现

### 5.1 分包加载策略

采用**主包 + 双分包**架构，优化首屏加载速度：

| 包 | 路径 | 页面数 | 说明 |
|----|------|--------|------|
| 主包 | `pages/` | 5 | 登录 / 首页 / 活动 / 购物车 / 我的 |
| 用户分包 | `user-pages/` | 18 | 所有用户功能页 |
| 员工分包 | `staff-pages/` | 2 | 员工工作台 + 核销页 |

### 5.2 订单自动取消机制

```
用户下单 → order-timer 启动 15min 倒计时 → 
  ├─ 用户主动支付 → 清除定时器 ✓
  └─ 超时未支付 → 调用取消接口 → 本地标记已取消 → UI 更新
```

### 5.3 多端角色适配

```
登录成功 → 后端返回 role → 存储 user_role → 
  ├─ user → custom-tab-bar 渲染 4-tab 用户界面
  └─ staff → custom-tab-bar 渲染 2-tab 员工界面 + 默认跳转工作台
```

### 5.4 权限控制

- **前端拦截层**：`api.js` 根据接口 URL 前缀判断是否需要 token，无 token 时拒绝请求并跳转登录
- **角色隔离**：员工页面（`staff-pages/`）只能通过员工角色访问
- **隐私权限**：已申请 `getLocation` 和 `chooseLocation` 权限，用于收货地址定位

## 六、项目亮点

1. **角色双模式**：同一套代码服务普通用户和员工，TabBar 和页面路由完全差异化
2. **分包优化**：18 个用户页面独立分包，主包精简至 5 页，启动速度快
3. **订单闭环**：从下单 → 支付 → 发货 → 完成 / 取消，全流程覆盖，含超时自动取消
4. **API 工程化**：统一的请求封装、错误处理、Loading 管理、401 自动跳转
5. **体验细节**：已取消订单差异化展示、订单实时计时器、物流轨迹追踪
