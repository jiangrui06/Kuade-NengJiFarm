## 一、项目概述

农业电商 + 体验式认购小程序，面向两类角色：

- **普通用户**：浏览商品、参与活动、认购一亩田、下单购物、查看物流
- **员工用户**：扫码核销活动券，管理核销记录

采用 **自定义 TabBar** 实现角色差异化导航——普通用户 4 个 Tab（首页 / 活动 / 购物车 / 我的），员工 2 个 Tab（工作台 / 我的），登录时根据 `user_role` 自动切换。

## 三、核心功能

### 3.1 用户体系

| 项目 | 说明 |
|------|------|
| **登录流程** | `wx.login()` 获取 code → 后端换 `session_key` → `getPhoneNumber` 获取手机号 |
| **接口** | `POST /api/Auth/wx-phone-login` |
| **角色标识** | `user_role` 存 Storage：`user`（普通用户）/ `staff`（员工） |
| **Token 管理** | 登录返回 token 存 Storage，请求头 `Authorization: Bearer <token>`，无 token 自动跳登录 |

### 3.2 商城 & 订单

购买链路：**商品列表 → 详情 → 加入购物车 → 确认订单 → 支付**

订单状态：`pending`（15 分钟倒计时）→ `paid` → `shipping` → `completed` / `cancelled`

- 三种订单类型：普通商城、活动、认购
- **订单计时器**：待付款倒计时，超时自动取消并 UI 更新
- **已取消订单**：隐藏地址和物流，仅保留商品信息 + 编号 + 取消时间

### 3.3 活动 & 认购

- 活动列表 → 详情 → 报名，支持**核销二维码**签到验券
- 认购一亩田：独立订单类型，走 `/api/acres` 接口系列

### 3.4 员工核销

| 页面 | 功能 |
|------|------|
| staff-home | 工作台首页，`wx.scanCode` 扫码入口 |
| staff-verify | 展示券码详情，执行核销（`POST /api/staff/verify`） |

## 四、API 接口

封装在 `utils/api.js`，共 **8 大模块、40+ 接口**：

| 模块 | 前缀 | 功能 |
|------|------|------|
| home | `/api/home` | 首页数据聚合 |
| goods | `/api/goods` | 商品 CRUD + 搜索 |
| activity | `/api/activity` | 活动列表 / 详情 / 报名 |
| acre | `/api/acres` | 一亩田认购 |
| order | `/api/orders` | 订单全生命周期 |
| cart | `/api/cart` | 购物车增删改查 |
| pay | `/api/pay` | 微信 JSAPI 支付 |
| logistics | `/api/logistics` | 物流轨迹 |
| staff | `/api/staff` | 扫码核销 |
| user | `/api/user` | 用户信息 / 地址 |
| file | `/api/file` | 图片上传下载 |

**规范**：`Authorization: Bearer <token>` | 响应 `{ code: 0, data: {...} }` | 401 自动跳登录

## 五、关键技术

- **分包加载**：主包 5 页 + 用户分包 18 页 + 员工分包 2 页，优化首屏速度
- **角色双模式**：同一代码库服务两类用户，TabBar 和路由完全差异化
- **自动取消**：下单 → 15 分钟倒计时 → 超时自动取消
- **权限控制**：前端按 URL 前缀拦截未认证请求；员工页面角色隔离；已申请 `getLocation` / `chooseLocation` 隐私权限
- **订单闭环**：下单 → 支付 → 发货 → 完成 / 取消，全流程覆盖
