# 订单聚合接口文档

## 1. 通用说明

### 1.1 基础地址

```text
http://你的后端地址
```

### 1.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | number | `0` 表示成功，其他表示失败 |
| message | string | 提示信息 |
| data | any | 业务数据 |

### 1.3 当前数据来源

`/api/orders` 是订单聚合接口，后端会从以下真实订单表聚合数据：

| 订单类型 | 数据表 | type |
| --- | --- | --- |
| 商品订单 | `commodity_orders` | `goods` |
| 点餐订单 | `dish_orders` | `food` |
| 活动订单 | `activity_orders` | `activity` |

> 当前数据库中没有 `orders` 总表，因此该接口不会查询 `orders` 表。

---

## 2. 获取订单列表

```http
GET /api/orders
```

### 2.1 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| type | string | 否 | 空 | 订单类型：`all`、`goods`、`food`、`activity` |
| status | string | 否 | 空 | 订单状态：`all`、`pending`、`paid`、`shipping`、`completed`、`cancelled` |
| keyword | string | 否 | 空 | 关键字，支持订单号或订单 ID 匹配 |
| page | number | 否 | `1` | 页码 |
| pageSize | number | 否 | `10` | 每页数量 |
| sortBy | string | 否 | `createTime` | 排序字段：`createTime`、`totalPrice`、`price` |
| sortOrder | string | 否 | `desc` | 排序方向：`asc`、`desc` |

### 2.2 响应 data

```json
{
  "orders": [
    {
      "id": "GOODS202604270001",
      "orderId": "1",
      "orderNumber": "GOODS202604270001",
      "type": "goods",
      "typeText": "商品",
      "status": "pending",
      "statusText": "Pending",
      "createTime": "2026-04-27 12:00:00",
      "totalPrice": 23.7,
      "totalQuantity": 3,
      "tableNumber": 0,
      "items": [
        {
          "id": "1",
          "name": "商品名称",
          "price": 8.9,
          "quantity": 2,
          "image": "http://host/api/file/image/xxx.jpg"
        }
      ]
    }
  ],
  "total": 1,
  "page": 1,
  "pageSize": 10,
  "totalPages": 1
}
```

### 2.3 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| orders | array | 订单列表 |
| total | number | 总数量 |
| page | number | 当前页码 |
| pageSize | number | 每页数量 |
| totalPages | number | 总页数 |

订单对象：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 前端展示用 ID，当前返回订单号 |
| orderId | string | 数据库订单 ID |
| orderNumber | string | 平台订单号 |
| type | string | `goods`、`food`、`activity` |
| typeText | string | 类型文本 |
| status | string | 订单状态 |
| statusText | string | 状态文本 |
| createTime | string | 创建时间 |
| totalPrice | number | 订单金额 |
| totalQuantity | number | 商品/菜品/活动数量 |
| tableNumber | number | 餐桌 ID，仅点餐订单可能有值 |
| items | array | 订单明细，列表接口默认返回前 2 条 |

## 3. 获取订单数量统计

```http
GET /api/orders/counts
```

### 3.1 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| type | string | 否 | 空 | 订单类型：`all`、`goods`、`food`、`activity` |

### 3.2 响应 data

```json
{
  "pending": 1,
  "paid": 2,
  "shipping": 1,
  "completed": 3,
  "cancelled": 0,
  "total": 7
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| pending | number | 待付款数量 |
| paid | number | 已付款/待发货数量 |
| shipping | number | 运输中数量，主要用于商品订单 |
| completed | number | 已完成数量 |
| cancelled | number | 已取消数量 |
| total | number | 总数量 |

---

## 4. 获取订单详情

```http
GET /api/orders/{id}
```

### 4.1 路径参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 支持订单号，如 `GOODS202604270001`；也支持数字订单 ID |

### 4.2 响应 data

```json
{
  "id": "GOODS202604270001",
  "orderId": "1",
  "orderNumber": "GOODS202604270001",
  "type": "goods",
  "typeText": "商品",
  "status": "pending",
  "statusText": "Pending",
  "createTime": "2026-04-27 12:00:00",
  "totalPrice": 23.7,
  "totalQuantity": 3,
  "tableNumber": 0,
  "shippingAddress": {
    "name": "张三",
    "phone": "13800000000",
    "address": "湖南省长沙市岳麓区xxx"
  },
  "items": [
    {
      "id": "1",
      "name": "商品名称",
      "price": 8.9,
      "quantity": 2,
      "image": "http://host/api/file/image/xxx.jpg"
    }
  ],
  "paymentMethod": null,
  "transactionId": null
}
```

### 4.3 字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 前端展示用 ID，当前返回订单号 |
| orderId | string | 数据库订单 ID |
| orderNumber | string | 平台订单号 |
| type | string | 订单类型 |
| typeText | string | 类型文本 |
| status | string | 订单状态 |
| statusText | string | 状态文本 |
| createTime | string | 创建时间 |
| totalPrice | number | 订单金额 |
| totalQuantity | number | 数量 |
| tableNumber | number | 餐桌 ID |
| shippingAddress | object/null | 收货地址，商品订单可能有值 |
| items | array | 完整订单明细 |
| paymentMethod | string/null | 支付方式 |
| transactionId | string/null | 微信支付单号 |

## 5. 订单状态映射

### 5.1 商品订单 `commodity_orders`

| order_status_id | status | 说明 |
| --- | --- | --- |
| 1 | pending | 待付款 |
| 2 | paid | 已付款/待发货 |
| 3 | shipping | 运输中 |
| 4 | completed | 已完成 |
| 5 | cancelled | 已取消 |

### 5.2 点餐/活动订单

| order_status_id | status | 说明 |
| --- | --- | --- |
| 1 | pending | 待付款 |
| 2 | paid | 已付款 |
| 3 | completed | 已完成/已核销 |
| 4 | cancelled | 已取消 |

---

## 6. 调用示例

### 6.1 查询全部订单

```http
GET /api/orders?page=1&pageSize=10
```

### 6.2 查询商品订单

```http
GET /api/orders?type=goods&page=1&pageSize=10
```

### 6.3 查询待付款订单

```http
GET /api/orders?status=pending&page=1&pageSize=10
```

### 6.4 查询订单详情

```http
GET /api/orders/GOODS202604270001
```
