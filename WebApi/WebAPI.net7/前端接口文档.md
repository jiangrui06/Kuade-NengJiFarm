## 能记农场小程序前端接口文档（后端 WebAPI.net7）

> 说明：所有接口返回统一结构：
>
> ```json
> {
>   "code": 0,
>   "message": "ok",
>   "data": { ... }
> }
> ```
>
> - `code = 0` 表示成功，非 0 为失败（400 参数错误、404 资源不存在、1000 登录失败等）。
> - 文档中的 URL 均为相对路径，前面需要拼接你的 API 域名 / 基础路径。

---

## 1. 认证相关接口（AuthController）

### 1.1 一键登录

- **URL**: `/api/auth/login`
- **Method**: `POST`
- **Body(JSON)**:

| 字段      | 类型   | 必填 | 说明                      |
|-----------|--------|------|---------------------------|
| deviceId  | string | 是   | 设备 ID，用作用户唯一标识 |
| platform  | string | 否   | 平台标识，如 `wx`         |
| version   | string | 否   | 小程序版本号              |

- **Response.data**:

```json
{
  "token": "string",
  "userInfo": {
    "id": 1,
    "nickname": "昵称",
    "avatar": "头像URL"
  }
}
```

### 1.2 微信登录

- **URL**: `/api/auth/wechat`
- **Method**: `POST`
- **Body(JSON)**:

| 字段          | 类型   | 必填 | 说明                         |
|---------------|--------|------|------------------------------|
| code          | string | 是   | `wx.login` 获取到的 code     |
| encryptedData | string | 否   | 预留字段，目前未使用         |
| iv            | string | 否   | 预留字段，目前未使用         |

- **Response.data**：同一键登录。

### 1.3 手机号登录

- **URL**: `/api/auth/phone`
- **Method**: `POST`
- **Body(JSON)**:

| 字段  | 类型   | 必填 | 说明                     |
|-------|--------|------|--------------------------|
| phone | string | 是   | 手机号                   |
| code  | string | 是   | 验证码（当前仅校验非空） |

- **Response.data**：同一键登录。

### 1.4 登出

- **URL**: `/api/auth/logout`
- **Method**: `POST`
- **Header**: 可选 `Authorization: Bearer {token}`（当前不校验）
- **Response.data**: `null`

### 1.5 检查登录状态

- **URL**: `/api/auth/check`
- **Method**: `GET`
- **Response.data**:

```json
{
  "isLoggedIn": true,
  "userInfo": {
    "id": 0,
    "nickname": "游客",
    "avatar": ""
  }
}
```

> 当前未真正解析 token，主要用于前端流程联调。

## 4. 商品相关接口（GoodsController & CommodityController）

### 4.1 商品分页列表

- **URL**: `/api/goods`
- **Method**: `GET`
- **Query**:

| 字段       | 类型   | 必填 | 说明              |
|------------|--------|------|-------------------|
| pageIndex  | int    | 否   | 页码，默认 1      |
| pageSize   | int    | 否   | 每页数量，默认 10 |
| categoryId | int    | 否   | 分类 ID           |
| keyword    | string | 否   | 名称模糊匹配关键字 |

- **Response.data**:

```json
{
  "pageIndex": 1,
  "pageSize": 10,
  "total": 100,
  "items": [
    {
      "id": "guid",
      "name": "商品名",
      "description": null,
      "price": 0,
      "imageUrl": null
    }
  ]
}
```

> 这里的 `id` 为占位 GUID，真正主键请使用 `commodity_id`（即其他接口中的 `id`）。

### 4.2 商品详情（文档主用）

- **URL**: `/api/goods/detail`
- **Method**: `GET`
- **Query**:

| 字段    | 类型 | 必填 | 说明           |
|---------|------|------|----------------|
| goodsId | int  | 是   | 商品主键 `id` |

- **Response.data**:

```json
{
  "id": 1,
  "name": "商品名",
  "price": 0,
  "image": "/api/commodity/1/image",
  "detailImage": "/api/commodity/1/image",
  "description": "规格描述",
  "weight": "",
  "storage": "",
  "stock": 100,
  "tags": []
}
```

### 4.3 Demo 商品详情（按路径参数）

- **URL**: `/api/goods/{id}`
- **Method**: `GET`
- **Path**:

| 字段 | 类型 | 说明                     |
|------|------|--------------------------|
| id   | int  | 商品主键 `commodity_id` |

- **Response.data**:

```json
{
  "id": "guid",
  "name": "商品名",
  "price": 0
}
```

### 4.4 首页推荐商品

- **URL**: `/api/goods/recommend`
- **Method**: `GET`
- **Response.data**: `GoodsDto[]`（结构同上，取最新 4 条商品）。

### 4.5 商品图片

- **URL**: `/api/commodity/{id}/image`
- **Method**: `GET`
- **说明**：
  - 如果数据库 `image_data` 有值：直接输出图片二进制内容（`image/jpeg` / `image/png`）。
  - 否则如果 `image_url` 不为空：302 重定向到该 URL。
  - 否则返回 404。

## 5. 购物车相关接口（CartController）

> 当前购物车未持久化到独立表，仅 `/list` 从商品表生成模拟数据，其余接口为成功占位，方便前端流程打通。

### 5.3 更新购物车商品数量

- **Method**: `PUT`

- **Response.data**: `null`

- **Body**: 可自定义。
- **Response.data**: `null`

- **URL**: `/api/cart/clear`

- **Response.data**: `null`

## 6. 订单相关接口（OrdersController）

### 6.1 创建订单

- **URL**: `/api/order/create`
- **Method**: `POST`
- **Body(JSON)**:

| 字段       | 类型     | 必填 | 说明                             |
|------------|----------|------|----------------------------------|
| addressId  | int      | 是   | 收货地址 ID                      |
| totalPrice | decimal  | 是   | 总价格                           |
| remark     | string   | 否   | 订单备注                         |
| cartIds    | int[]    | 否   | 购物车项 ID 列表（当前未使用）   |

- **Response.data**:

```json
{ "orderId": 1 }
```

### 6.2 获取订单列表

- **URL**: `/api/order/list`
- **Method**: `GET`
- **Query**:

| 字段     | 类型   | 必填 | 说明                                   |
|----------|--------|------|----------------------------------------|
| status   | string | 否   | `all` 或其它，非 `all` 时示例过滤非 0 状态 |
| page     | int    | 否   | 页码，默认 1                           |
| pageSize | int    | 否   | 每页数量，默认 10                      |

- **Response.data**:

```json
{
  "orderList": [
    {
      "id": 1,
      "orderNo": "202603110001",
      "totalPrice": 69.99,
      "status": "0",
      "createTime": "2026-03-11 10:00:00"
    }
  ],
  "total": 1,
  "page": 1,
  "pageSize": 10
}
```

### 6.3 获取订单详情

- **URL**: `/api/order/detail`
- **Method**: `GET`
- **Query**:

| 字段    | 类型 | 必填 | 说明    |
|---------|------|------|---------|
| orderId | long | 是   | 订单 ID |

- **Response.data**:

```json
{
  "id": 1,
  "orderNo": "202603110001",
  "totalPrice": 69.99,
  "status": "0",
  "createTime": "2026-03-11 10:00:00",
  "address": {
    "name": "收货人",
    "phone": "手机号",
    "address": "详细地址"
  },
  "goodsList": []
}
```

> `goodsList` 当前为空数组，后续如接入订单明细表，可在此处补充商品信息。

### 6.4 取消订单

- **URL**: `/api/order/cancel`
- **Method**: `PUT`
- **Body(JSON)**:

| 字段    | 类型 | 必填 | 说明    |
|---------|------|------|---------|
| orderId | long | 是   | 订单 ID |

- **行为**：将 `order_status` 置为 `4`（示例“已取消”）。
- **Response.data**: `null`

### 6.5 确认收货

- **URL**: `/api/order/{id}/confirm`
- **Method**: `POST`
- **Path**:

| 字段 | 类型 | 说明    |
|------|------|---------|
| id   | long | 订单 ID |

- **行为**：将 `order_status` 置为 `3`（示例“已完成”）。
- **Response.data**: `null`

## 8. 备注

- 当前所有接口已对接 MySQL 数据库的表结构（用户、分类、商品、订单、收货地址等），但个别字段（价格、标签、订单明细等）仍为占位实现，后续可以在不改动接口形状的前提下逐步完善。
- 登录相关接口目前使用简单 Base64 token，仅用于前联调；生产请替换为 JWT 等安全方案，并在各业务接口中验证 `Authorization`。
