# 能记农场 — API 接口文档

> 本文档记录本次新增/修改的接口。原有接口未变化的不再赘述。

## 一、配送方式选择（P0）

### 1.1 创建商品订单

**`POST /api/commodity-order/create`**

**修改说明：** 新增 `deliveryMethod` 字段，支持快递/自取两种配送方式。

**Request Body:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| addressId | int | 快递必填 | 收货地址ID，自取时传0或不传 |
| deliveryMethod | string | 否 | `express`（快递，默认）/ `pickup`（自取） |
| items | array | 是 | 商品列表 |

```json
{
  "addressId": 1,
  "deliveryMethod": "express",
  "items": [
    { "id": "1", "quantity": 2, "price": 30 }
  ]
}
```

**Response (express):**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "GOODS20260517123456789123",
    "orderId": "GOODS20260517123456789123",
    "orderNumber": "GOODS20260517123456789123",
    "orderType": "goods",
    "status": "pending",
    "deliveryMethod": "express",
    "verifyCode": null,
    "totalPrice": 60.00,
    "createTime": "2026-05-17 12:34:56"
  }
}
```

**Response (pickup):**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "GOODS20260517123456789123",
    "orderId": "GOODS20260517123456789123",
    "orderNumber": "GOODS20260517123456789123",
    "orderType": "goods",
    "status": "pending",
    "deliveryMethod": "pickup",
    "verifyCode": "A3B7X9K2M5P8",
    "totalPrice": 60.00,
    "createTime": "2026-05-17 12:34:56"
  }
}
```

## 二、支付回调改造（积分入账 + 自取状态）

### 2.1 微信支付回调

**`POST /api/pay/notify`**（内部接口，微信服务器调用）

**修改说明：** 支付成功后：
- 自取商品订单 → 状态设为 **8（待核销）**
- 快递商品订单 → 状态设为 **2（待发货）**（不变）
- 所有订单 → 自动积分入账 **消费金额÷10（10元=1积分）**

## 三、员工核销（P1）

### 3.1 核销券码

**`POST /api/staff/verify`**

**修改说明：** 现在同时支持核销两种券码：
1. **商品自取核销码**（12位字母数字，如 `A3B7X9K2M5P8`）
2. **活动/采摘券券码**（OrderNo 或 OrderId）

**Request Body:**
```json
{
  "code": "A3B7X9K2M5P8"
}
```

**Response — 商品自取核销成功（首次核销）：**
```json
{

  "message": "核销成功",
  "data": {
    "verified": true,
    "alreadyVerified": false,

    "typeName": "商品自取",
    "userName": "张三",
    "userPhone": "138****5678",
    "content": "到店自取商品",
    "title": "商品自取",
    "orderNo": "GOODS20260517123456789123",
    "verifyTime": "2026-05-17 14:30:00",
    "participantCount": 3,
    "count": 3
  }
}
```

**Response — 商品自取已核销：**
```json
{
  "code": 200,
  "message": "",
  "data": {
    "verified": true,
    "alreadyVerified": true,
    "voucherType": "goods_pickup",
```

## 四、积分商城（P2）

### 4.1 获取积分总览

**`GET /api/points/summary`**

**Headers:** `Authorization: Bearer {token}`

**Response:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalPoints": 150,
    "earnedPoints": 200,
    "spentPoints": 50,
    "todayEarned": 30
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| totalPoints | int | 当前可用积分 |
| earnedPoints | int | 累计获得积分 |
| spentPoints | int | 累计消费积分 |
| todayEarned | int | 今日获得积分 |

### 4.4 积分兑换商品

**`POST /api/points/exchange`**

**Headers:** `Authorization: Bearer {token}`

**Request Body:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| commodityId | int | 是 | 商品ID |
| quantity | int | 否 | 数量，默认1 |

```json
{
  "commodityId": 1,
  "quantity": 1
}
```

**Response:**
```json
{
  "code": 200,
  "message": "兑换成功",
  "data": {
    "exchangeId": 0,
    "orderNo": "EXC20260517123456789123",
    "pointsSpent": 50,
    "pointsRemaining": 100,
    "status": "completed"
  }
}
```

**错误响应（积分不足）：**
```json
{
  "code": 409,
  "message": "积分不足"
}
```

**错误响应（库存不足）：**
```json
{
  "code": 409,
  "message": "商品库存不足"
}
```

**错误响应（商品不支持兑换）：**
```json
{
  "code": 400,
  "message": "该商品不支持积分兑换"
}
```

### 4.7 手动积分入账（管理员用）

**Request Body:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

```json
{
  "code": 200,
  "message": "积分入账成功",
  "data": null
}
```

## 五、状态映射

### 商品订单状态

| 状态码 | 名称 | 配送方式 | 说明 |
|--------|------|----------|------|
| 1 | 待付款 | 通用 | 未支付 |
| 2 | 待发货 | 快递 | 已支付，等待发货 |
| 3 | 运输中 | 快递 | 已发货 |
| 4 | 已完成 | 通用 | 已签收或已核销 |
| 5 | 已取消 | 通用 | 已取消 |
| 6 | 退款中 | 通用 | 退款处理中 |
| 7 | 已退款 | 通用 | 已退款 |
| **8** | **待核销** | **自取** | **已支付，等待到店核销** |
| **9** | **已核销** | **自取** | **已到店核销完成** |

### 积分规则

- 在场消费 **10元 = 1积分**
- 仅计算支付成功后的消费金额
- 支付回调中自动入账
- 积分可用于兑换积分商品（`points_price > 0` 的商品）
