# 后台用户登录+管理 API 接口文档

本文档用于明确区分两套账号体系，避免把普通员工和普通用户混在同一套接口里。

## 一、接口拆分原则

- 后台账号体系：管理员、普通员工
  统一走 `/api/back-user/*`
- 普通用户体系：面向小程序、H5、客户端用户
  统一走 `/api/app-user/*`
- 两套账号不要共用登录接口、不要共用新增编辑接口、不要混用角色字段

## 二、时间格式约定

所有时间字段统一使用 `yyyy-MM-dd HH:mm` 格式。

- 正确示例：`2026-04-13 09:30`
- 正确示例：`2026-12-01 17:05`
- 不再使用：`2026/04/13 09:30`
- 不再使用：`2026-4-13 09:30`

## 三、公共响应结构

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | Number | 状态码，`200` 表示成功 |
| `message` | String | 提示文案 |
| `data` | Object / Array / Null | 业务数据 |

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 四、后台账号体系：管理员 / 普通员工

### 4.1 说明

- 本节只处理后台可登录账号
- 角色只允许：`管理员`、`普通员工`
- 不允许把 `普通用户` 放进后台账号接口返回结果里

### 4.2 接口 1：后台账号登录

| 项目 | 内容 |
|---|---|
| 接口名称 | 后台账号登录 |
| 请求路径 | `/api/back-user/login` |
| 请求方式 | `POST` |
| 请求格式 | `application/json` |
| 接口说明 | 管理员和普通员工登录后台管理系统 |

#### 请求参数

| 字段 | 是否必传 | 类型 | 示例 | 说明 |
|---|---|---|---|---|
| `id` | 是 | String | `staff_10001` | 后台登录账号 |
| `password` | 是 | String | `123456` | 登录密码 |

#### 请求示例

```json
{
  "id": "staff_10001",
  "password": "123456"
}
```

#### 成功响应示例

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "staff_10001",
    "nickname": "李四",
    "gender": "男",
    "role": "管理员",
    "status": "启用",
    "loginTime": "2026-04-13 09:30",
    "token": "back-user-token-20260413-xxxx"
  }
}
```

#### 失败场景

| 场景 | 状态码 | 提示文案 |
|---|---|---|
| 账号为空 | `400` | 账号不能为空 |
| 密码为空 | `400` | 密码不能为空 |
| 账号不存在 | `401` | 账号不存在 |
| 密码错误 | `401` | 密码错误，请重新输入 |
| 账号禁用 | `403` | 账号已禁用，请联系管理员 |

## 五、普通用户体系：普通用户

### 5.1 说明

- 本节只处理普通用户
- 角色只允许：`普通用户`
- 普通用户不要接到 `/api/back-user/*`
- 后台普通员工也不要接到 `/api/app-user/*`

### 5.2 接口 1：普通用户登录

| 项目 | 内容 |
|---|---|
| 接口名称 | 普通用户登录 |
| 请求路径 | `/api/app-user/login` |
| 请求方式 | `POST` |
| 接口说明 | 普通用户登录小程序、H5 或客户端 |

#### 请求参数

| 字段 | 是否必传 | 类型 | 示例 | 说明 |
|---|---|---|---|---|
| `phone` | 是 | String | `12242364893` | 用户手机号 |
| `password` | 是 | String | `123456` | 登录密码 |

#### 成功响应示例

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "app_user_20001",
    "phone": "12242364893",
    "nickname": "张三",
    "gender": "男",
    "role": "普通用户",
    "status": "启用",
    "loginTime": "2026-04-13 09:35",
    "token": "app-user-token-20260413-xxxx"
  }
}
```

### 5.3 普通用户接口边界

- 普通用户登录、注册、资料维护建议统一放在 `/api/app-user/*`
- 普通用户不进入后台员工新增、编辑、删除接口
- 如果后续需要普通用户管理，请单独设计 `/api/app-user/list`、`/api/app-user/detail`、`/api/app-user/changeStatus`

## 六、通用请求头

后台管理接口建议统一要求以下请求头：

| 字段 | 是否必传 | 类型 | 说明 |
|---|---|---|---|
| `Content-Type` | 是 | String | 固定为 `application/json` |
| `token` | 是 | String | 登录接口返回的令牌 |

## 七、通用失败场景

| 场景 | 状态码 | 提示文案 |
|---|---|---|
| token 无效或过期 | `401` | 登录已过期，请重新登录 |
| 权限不足 | `403` | 权限不足，仅管理员可操作 |
| 参数错误 | `400` | 请求参数不完整或格式错误 |
| 数据不存在 | `404` | 用户不存在 |
| 服务异常 | `500` | 服务器异常，请稍后重试 |

## 八、落地建议

- 后台页面登录统一接 `/api/back-user/login`
- 普通用户登录统一接 `/api/app-user/login`
- 普通员工和普通用户的数据表建议分表或至少分类型隔离
- 文档、接口、前端字段都统一使用 `yyyy-MM-dd HH:mm` 时间格式
