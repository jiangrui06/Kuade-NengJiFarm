# 后台管理员登录 + 员工/用户管理 API 接口文档

本文档用于后台管理系统账号与用户管理，分为两部分：

- 管理员账号仅用于后台登录
- 普通员工和普通用户在后台统一管理展示

## 一、接口拆分原则

- 管理员登录：单独用于后台登录，仅管理员可登录后台
- 用户管理：普通员工和普通用户统一管理，管理员不纳入用户管理列表
- 普通员工和普通用户通过 `type` 字段筛选

## 二、时间格式约定

页面当前统一按 `xxxx年xx月xx日 xx:xx` 展示时间。

示例：

- `2026年04月13日 09:30`
- `2026年12月01日 18:20`

## 三、管理员登录（后台登录）

### 3.1 接口信息

| 项目   | 内容                     |
| ---- | ---------------------- |
| 接口名称 | 管理员登录               |
| 请求路径 | `/api/back-user/login` |
| 请求方式 | `POST`                 |
| 请求格式 | `application/json`     |
| 接口说明 | 管理员单独管理，仅用于后台登录        |

### 3.2 请求参数

| 字段         | 是否必传 | 类型     | 示例           | 说明     |
| ---------- | ---- | ------ | ------------ | ------ |
| `id` | 是    | String | `staff10001` | 管理员账号 |
| `password` | 是    | String | `123456`     | 登录密码   |

### 3.3 请求示例

```json
{
  "id": "staff10001",
  "password": "123456"
}
```

### 3.4 成功响应示例

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "staff_10001",
    "account": "staff10001",
    "role": "管理员",
    "loginTime": "2026年04月13日 09:30",
    "token": "back-user-token-20260413-xxxx"
  }
}
```

### 3.5 失败场景

| 场景    | 状态码   | 提示文案         |
| ----- | ----- | ------------ |
| 账号不存在 | `404` | 账号不存在        |
| 密码错误  | `401` | 密码错误         |
| 非管理员  | `403` | 无权限，请使用管理员账号登录 |

## 四、用户列表（普通员工和普通用户统一管理）

### 4.1 接口信息

| 项目   | 内容                     |
| ---- | ---------------------- |
| 接口名称 | 用户列表                 |
| 请求路径 | `/api/user/list` |
| 请求方式 | `GET`                  |
| 接口说明 | 后台统一管理普通员工和普通用户，管理员不在该列表中 |

### 4.2 Query 参数

| 字段         | 是否必传 | 类型     | 示例              | 说明                |
| ---------- | ---- | ------ | --------------- | ----------------- |
| `pageNum`  | 是    | Number | `1`             | 页码                |
| `pageSize` | 是    | Number | `15`            | 每页条数              |
| `type`     | 否    | String | `员工` / `用户` | 筛选普通员工或普通用户，不传则返回全部  |
| `keyword`  | 否    | String | `staff`         | 按账号/昵称模糊搜索        |

### 4.4 分页字段说明

| 字段         | 类型     | 说明                   |
| ---------- | ------ | -------------------- |
| `records`  | Array  | 数据列表               |
| `total`    | Number | 总记录数                |
| `pageNum`  | Number | 当前页码，从1开始         |
| `pageSize` | Number | 每页条数                |
| `pages`    | Number | 总页数                 |

### 4.5 员工字段说明

| 字段         | 类型     | 示例              | 说明                |

| `loginTime` | String | `2026年04月12日 18:20` | 最后登录时间            |

## 九、落地建议

1. **管理员登录**：管理员账号单独管理，后台登录接口使用 `/api/back-user/login`
2. **统一管理列表**：普通员工和普通用户在后台统一管理展示，可通过 `type` 参数筛选
3. **管理员隔离**：管理员不显示在用户列表中，也不走普通用户管理相关接口
4. **时间格式**：统一使用 `xxxx年xx月xx日 xx:xx` 格式
5. **登录验证**：登录时验证权限，仅管理员可登录后台
6. **员工默认值**：员工没有手机号和头像时显示"无"
