# 后台用户登录+管理 API 接口文档

本文档用于后台管理系统用户管理，包含后台普通员工（后台）和普通用户（小程序、客户端）的统一管理。

## 一、接口拆分原则

- 用户管理：员工和用户统一走 `/api/user/*`
- 员工和用户通过 `type` 字段筛选

## 二、时间格式约定

页面当前统一按 `xxxx年xx月xx日 xx:xx` 展示时间。

示例：

- `2026年04月13日 09:30`
- `2026年12月01日 18:20`

## 三、用户登录

### 3.1 接口信息

| 项目   | 内容                     |
| ---- | ---------------------- |
| 接口名称 | 用户登录                 |
| 请求路径 | `/api/user/login` |
| 请求方式 | `POST`                 |
| 请求格式 | `application/json`     |
| 接口说明 | 管理员登录（仅管理员可登录）        |

### 3.2 请求参数

| 字段         | 是否必传 | 类型     | 示例           | 说明     |
| ---------- | ---- | ------ | ------------ | ------ |
| `account` | 是    | String | `staff10001` | 登录账号 |
| `password` | 是    | String | `123456`     | 登录密码   |

### 3.3 请求示例

```json
{
  "account": "staff10001",
  "password": "123456"
}
```

### 3.4 成功响应示例

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "staff_10001",
    "account": "staff10001",
    "role": "管理员",
    "loginTime": "2026年04月13日 09:30",
    "token": "back-user-token-20260413-xxxx"
  }
}
```

### 3.5 失败场景

| 场景    | 状态码   | 提示文案         |
| ----- | ----- | ------------ |
| 账号不存在 | `404` | 账号不存在        |
| 密码错误  | `401` | 密码错误         |
| 非管理员  | `403` | 无权限，请使用管理员账号登录 |

## 十、落地建议

1. **接口路径**：所有接口统一走 `/api/user/*`
2. **员工和用户**：通过 `type` 参数筛选（传`员工`或`用户`）
3. **状态字段**：使用数字表示，1=启用，0=禁用
4. **时间格式**：统一使用 `xxxx年xx月xx日 xx:xx` 格式
5. **管理员**：管理员不显示在用户列表中
6. **登录验证**：登录时验证权限，仅管理员可登录
7. **员工默认值**：员工没有手机号和头像时显示"无"
