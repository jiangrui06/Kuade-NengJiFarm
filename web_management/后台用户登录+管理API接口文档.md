# 用户登录+管理API接口文档

本文档基于前端默认用户数据（`getDefaultUsers()`）编写，适用于前后端对接、mock接口开发，字段完全匹配现有用户结构，无额外冗余信息

# 第一部分：用户登录接口

## 一、接口基础信息

|参数项|详细说明|
|---|---|
|接口名称|用户账号登录|
|请求路径|`/api/user/login`|
|请求方式|POST|
|请求格式|application/json|
|接口描述|通过手机号+密码校验登录，校验账号状态，返回用户核心信息与登录凭证|
## 二、请求参数说明

### 2.1 请求头（Headers）

|参数名|是否必传|参数类型|说明|
|---|---|---|---|
|Content-Type|是|String|固定值：application/json，标识请求体为JSON格式|
### 2.2 请求体（Body）

|参数名|是否必传|参数类型|示例值|参数说明|
|---|---|---|---|---|
|phone|是|String|13758428019|用户手机号，11位纯数字，匹配用户数据中phone字段|
|password|是|String|123456|登录密码，测试环境统一默认密码：123456|
### 2.3 请求示例（JSON）

```json
{
  "phone": "18660478321",
  "password": "123456"
}
```

## 三、响应参数说明

### 3.1 公共响应结构

所有响应统一返回格式，包含状态码、提示信息、业务数据，便于前端统一处理

|参数名|参数类型|说明|
|---|---|---|
|code|Number|响应状态码：200=成功，400=参数错误，401=校验失败，403=账号禁用，500=服务器异常|
|message|String|响应提示文案，前端可直接展示|
|data|Object/Null|成功时返回用户数据，失败时为null|
### 3.2 成功响应（code=200）

登录成功后返回用户核心字段，与`getDefaultUsers()`数据结构一一对应

|参数名|参数类型|示例值|说明|
|---|---|---|---|
|id|String|wxid_li_si_001|客户微信号，作为用户唯一标识|
|phone|String|18660478321|登录手机号|
|nickname|String|李四|用户昵称|
|gender|String|男|用户性别（男/女）|
|role|String|管理员|用户角色（管理员/普通用户）|
|status|String|启用|账号状态（启用/禁用）|
|token|String|login-token-20260318|登录凭证，后续接口需携带|
### 3.3 成功响应示例（JSON）

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": "wxid_li_si_001",
    "phone": "18660478321",
    "nickname": "李四",
    "gender": "男",
    "role": "管理员",
    "status": "启用",
    "token": "login-token-20260318-xxxx"
  }
}
```

## 四、失败响应场景与示例

|失败场景|状态码|提示信息|响应示例|
|---|---|---|---|
|手机号/密码为空|400|手机号和密码不能为空|`{"code":400,"message":"手机号和密码不能为空","data":null}`|
|手机号格式错误|400|请输入正确的11位手机号|`{"code":400,"message":"请输入正确的11位手机号","data":null}`|
|手机号不存在|401|该手机号未注册|`{"code":401,"message":"该手机号未注册","data":null}`|
|密码错误|401|密码错误，请重新输入|`{"code":401,"message":"密码错误，请重新输入","data":null}`|
|账号已禁用|403|账号已禁用，请联系管理员|`{"code":403,"message":"账号已禁用，请联系管理员","data":null}`|
## 五、业务校验规则（核心）

- **账号匹配**：仅校验请求手机号与`getDefaultUsers()`数组中的phone字段一致

- **用户标识**：响应中的`id`字段统一使用客户微信号，作为用户唯一标识

- **密码校验**：测试环境统一密码为`123456`，无需加密，直接比对字符串

- **状态拦截**：用户status为**禁用**时，直接拒绝登录，返回403状态码

- **登录时间**：登录成功后，可同步更新对应用户的loginTime为当前系统时间

- **角色区分**：返回role字段，前端可根据角色（管理员/普通用户）展示不同页面权限

# 第二部分：用户管理接口（管理员权限）

权限说明：以下接口仅**管理员角色**可调用，请求头需携带登录返回的token凭证；所有接口沿用公共响应格式

## 一、公共请求头（所有管理接口必传）

|参数名|是否必传|参数类型|说明|
|---|---|---|---|
|Content-Type|是|String|固定值：application/json|
|token|是|String|登录接口返回的token凭证；当token过期、失效或已变更时，前端需自动提示并重新登录|

### 1.1 token失效自动处理约定

- **自动提示场景**：当管理接口返回`401`，或前端检测到当前页面保存的token与最新登录token不一致时，视为登录状态失效

- **统一提示文案**：建议前端统一提示`登录已过期或账号已在其他端重新登录，请重新登录`

- **统一处理动作**：清空本地登录信息（如`token`、`userInfo`），自动跳转回登录页
