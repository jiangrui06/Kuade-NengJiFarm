## 一、Bug 修复

### 1.1 `product-add.html` — 产品新增页面空白

- **问题**：Vue `methods` 对象中第 691-693 行有一段游离的赋值代码 `this.productForm = productData;`，没有函数名包裹，导致 Vue 解析 `methods` 失败，页面完全空白。
- **修复**：删除这 3 行代码。
- **状态**：已修复。

### 1.2 `order-product-detail.html` — 订单详情页面空白

- **问题**：Vue `methods` 对象中第 838-844 行有两个游离的 `return` 语句（`return {}` 和 `return null`），没有函数名包裹，在 `buildOrderItems` 和 `loadOrderDetail` 方法之间，导致 Vue 解析失败。
- **修复**：删除这两个游离的 `return` 语句块。
- **状态**：已修复。

### 1.3 `order-coupon.html` — 核销功能只改了前端状态，未调用后端 API

- **问题**：`confirmWriteOffCouponOrder()` 方法只修改了前端的 `orderStatus` 和 `verificationRecords`，没有调用 `POST /api/activity-order/verify` 接口。刷新页面后核销状态会丢失。
- **修复**：重写 `confirmWriteOffCouponOrder()` 方法：
  1. 先调用 `POST /api/activity-order/verify` 发送核销请求（body: `{ orderId, verifyCode }`）
  2. 后端返回成功后再更新前端本地状态
  3. 失败时弹出错误提示
- **状态**：已修复。

### 1.4 `order-coupon.html` — CSS 语法错误

- **问题**：第 501 行 Vue 模板的 `style` 属性中包含 `last-child:border-bottom:none;`，这是 CSS 语法，在 Vue 的 `style` 属性中无效。
- **修复**：改用 Vue 的 `:style` 绑定，通过 `v-for` 的 `index` 判断是否为最后一个元素，动态控制 `border-bottom`。
- **状态**：已修复。

### 1.5 `order-dish-detail.html` — 退款按钮无点击事件

- **问题**：页面中有两个「退款」按钮（标题栏一个、每行一个），都没有绑定 `@click` 事件，点击无任何反应。
- **修复**：
  1. 为两个按钮添加 `@click="openDishRefundModal"` 事件绑定
  2. 添加退款提示弹窗（模态框），说明后端菜品订单接口暂未开放退款功能
  3. 添加 `dishRefundModalVisible` 数据属性和 `openDishRefundModal` 方法
- **备注**：因 `DishOrderController` 后端没有退款接口，所以暂时只显示提示信息，待后端添加退款接口后可无缝接入。
- **状态**：已修复。

### 2.2 菜品管理增加批量删除

- **问题**：`dish.html` 已有全选 checkbox 和批量选择 UI，但删除时逐个调用单条删除 API，未使用批量删除接口。
- **修复**：

- **状态**：已修复。

### 2.3 用户管理对接角色列表 API

- **问题**：`user-add.html` 和 `user-edit.html` 的角色选择下拉框使用硬编码值（"工作人员"、"厨师"），未从后端动态获取。
- **修复**：
  1. 添加 `ROLES_LIST_API` 常量指向 `GET /api/back-user/roles/list`
  2. 添加 `roleList` 数据属性和 `loadRoles()` 方法
  3. 在 `mounted()` 中调用 `loadRoles()` 加载角色列表
  4. 角色下拉框改为 `v-for="role in roleList"` 动态渲染
- **状态**：已修复。

### 2.4 侧边栏菜单统一

- **问题**：部分页面将菜品管理显示为「点餐管理」，部分显示为「菜品管理」，存在不一致。

## 三、新增功能

### 3.1 员工核销管理页面

- **文件**：`staff.html`
- **功能**：
  1. **今日统计** — 调用 `GET /api/staff/today-stats` 展示核销统计数据（今日核销数、成功数、失败数、金额）
  2. **核销操作区** — 输入订单号和核销码，调用 `POST /api/staff/verify` 进行核销
  3. **券列表** — 调用 `GET /api/staff/vouchers` 展示可核销的券列表，支持关键词搜索和状态筛选
  4. **核销记录** — 调用 `GET /api/staff/verify-history` 展示历史核销记录，支持分页
- **状态**：已完成。

### 3.2 厨房管理页面

- **文件**：
  - `kitchen-login.html` — 厨房独立登录页面
  - `kitchen.html` — 厨房管理主页面
- **功能**：
  1. **厨房登录** — 调用 `POST /api/kitchen/login`，登录后跳转到主页面
  2. **今日统计** — 调用 `GET /api/kitchen/today-statistics` 展示总订单、已完成、待处理、已取消统计
  3. **订单列表** — 调用 `GET /api/kitchen/order/list`，支持按状态筛选
  4. **完成/取消菜品** — 调用 `POST /api/kitchen/dish/finish` 和 `POST /api/kitchen/dish/cancel`
  5. **订单详情弹窗** — 点击订单行查看详情，并可在弹窗中对单个菜品进行操作
  6. **退出登录** — 调用 `POST /api/kitchen/logout`
- **状态**：已完成。
